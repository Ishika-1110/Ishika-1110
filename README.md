<details>
<summary><strong>⬡ &nbsp; EDA — Insurance Dataset Analysis</strong></summary>

<br/>

> An end-to-end Exploratory Data Analysis project on a real-world Kaggle insurance dataset, uncovering the statistical drivers of medical charges across demographic and lifestyle features. Combines rigorous statistical analysis, professional visualization, and feature engineering to extract actionable insights from raw tabular data.

<br/>

| Attribute | Details |
|:---|:---|
| **Stack** | Python · Pandas · NumPy · Matplotlib · Seaborn |
| **Scale** | 1,338 records · 7 features · 4 categorical + 3 continuous variables |
| **Performance** | 10+ visualizations generated · 3 statistically significant correlations surfaced |
| **Security** | Offline analysis · no PII exposure |
| **Impact** | Identified smoking status as the strongest charge predictor (~3.8× cost increase) · 2 interaction features engineered to reduce multicollinearity |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-7C3AED?style=flat-square&logo=github&logoColor=white)](https://github.com/Ishika-1110/EDA-PROCESSS) |

<br/>

This project follows a disciplined EDA methodology: starting from raw data profiling and null/outlier detection, moving through univariate and bivariate distribution analysis, and concluding with feature engineering decisions informed by correlation heatmaps and pairplots. The analysis surfaces that smoking status alone accounts for a ~3.8× elevation in insurance charges — a finding with direct implications for actuarial modeling and risk segmentation. Two interaction features were subsequently engineered, reducing multicollinearity and improving downstream model interpretability.

<br/>

</details>
